Veloserver: Velocity Data Visualization Services

Description

Veloserver is a geospatial data visualization server designed to dynamically return velocity data such as winds and ocean currents from weather models and other datasets. It returns the data in formats optimized for visualizing in a web client including animated vector streamlines via gribjson. It includes built-in caching to reduce requests to external data sources and deliver quick results.

Usage

Default server port is 8104. Data is fetched on demand from the upstream model, converted to the requested format, cached, and returned. A trailing / is optional on every route.

Most requests are structured in the form:

/<model>/<product>/<format>/<datetime>/<projwin>

product and projwin are optional, which gives the following routes:

RouteReturns
/<model>/<format>/<datetime>Global data for the default product (winds)
/<model>/<format>/<datetime>/<projwin>Data subset to a bounding box
/<model>/<product>/<format>/<datetime>Global data for a specific product
/<model>/<product>/<format>/<datetime>/<projwin>A specific product, subset to a bounding box

There is also one dedicated route for Cloud-Optimized GeoTIFFs, which is HRRR only and takes no model or format: /cog/<product>/<datetime> returns an EPSG:3857 Cloud-Optimized GeoTIFF.

Parameters

model - hrrr (CONUS), gfs (global), or ecmwf (global; requires a licence and ~/.ecmwfapirc).

product - HRRR only; defaults to winds. Options: winds (10 m U/V vectors, the velocity layer), temp_2m, dewpoint_2m, rh_2m, wind_gust, pbl_height, precip_rate, smoke_massden.

format - gribjson (U/V vectors for animated streamlines, winds only), geotiff, or png. Only winds supports gribjson; for any other product use geotiff, png, or the /cog route (gribjson otherwise returns 400). gfs and ecmwf return gribjson only.

datetime - ISO 8601, e.g. 2025-01-01T06:00:00Z. A bare date means midnight. Each model rounds to its own run cadence: HRRR hourly, GFS every 6 hours, ECMWF 00z.

projwin - Bounding box ulx,uly,lrx,lry (west, north, east, south). Leave it out of the route for the full grid.

fxx is an optional query parameter for HRRR requests. It sets the forecast hour, which is how far ahead of the model run you want to look, so adding ?fxx=6 gives you the six-hour forecast. If you leave it off you get F00, which is called the analysis hour. The analysis hour is the model's best picture of the atmosphere at the moment the run started, rather than a forecast of some later time. Every run reaches F18, and the runs at 00, 06, 12, and 18z reach all the way out to F48.

Sample Requests

Winds (velocity / streamline layer)

Global HRRR winds: http://localhost:8104/hrrr/gribjson/2025-01-01T00:00:00Z

HRRR winds over Southern California: http://localhost:8104/hrrr/winds/gribjson/2025-01-01T00:00:00Z/-118,34,-117,33

Global GFS winds: http://localhost:8104/gfs/gribjson/2025-01-01T00:00:00Z

ECMWF winds over a box (06:00 UTC): http://localhost:8104/ecmwf/gribjson/2025-01-01T06:00:00Z/-118,34,-117,33

HRRR forecast hours (fxx)

Analysis (F00), the default: http://localhost:8104/hrrr/gribjson/2025-01-01T12:00:00Z

6 hours ahead of the 09z run: http://localhost:8104/hrrr/gribjson/2025-01-01T09:00:00Z?fxx=6

48 hours ahead of the 00z extended run: http://localhost:8104/hrrr/gribjson/2025-01-01T00:00:00Z?fxx=48

HRRR rasters (png, geotiff)

2 m temperature PNG: http://localhost:8104/hrrr/temp_2m/png/2025-01-01T00:00:00Z

PBL height GeoTIFF: http://localhost:8104/hrrr/pbl_height/geotiff/2025-01-01T00:00:00Z

Smoke during the Jan 2025 LA fires: http://localhost:8104/hrrr/smoke_massden/png/2025-01-08T21:00:00Z

Cloud-Optimized GeoTIFFs (/cog)

Winds COG: http://localhost:8104/cog/winds/2025-01-01T00:00:00Z

Temperature COG, 12-hour forecast: http://localhost:8104/cog/temp_2m/2025-01-01T00:00:00Z?fxx=12
